Split View in Sidebar

There are a few different ways Split View could work in Sidebar:

1. Fused Tabs

What we had in Honolulu: one tab with two pages in it.

Split tabs are created by:

  • Dragging two tabs together (this will be difficult in Sidebar because dragging is already a primary gesture. It'll be difficult to move things between lists or reorder them without accidentally fusing tabs)
  • Using a 'split' button to add another webview onto an existing tab
  • Right-clicking a link to open it in another pane within the same tab

Use cases where it works great:

  • A long-running task involving exactly two websites. For example, writing an essay based on an outline, or comparing two documents
  • A master/detail interaction, e.g. viewing a feed or news aggregator in the left pane while viewing content in the right pane

Use cases where it doesn't work great:

  • Tasks where you want a persistent sidebar on one side and many different tabs on the other. For example, using one pane as a scratchpad while you go about your day, or watching a video while working

2. Persistent Pane

What we have today (Sept. 2020): you pick one tab and make it a persistent split view pane, which sticks around while you switch tabs. Your current tab is displayed in the other pane.

Entry points:

  • Dragging a tab into the right pane of the browser
  • right-clicking and choosing 'Open in Split View', which would make the current page into a persistent sidebar (it may be confusing that the sidebar is persistent)

Use cases where it works great:

  • Tasks where you want a persistent sidebar on one side and many different tabs on the other. For example, using one pane as a scratchpad while you go about your day, or watching a video while working

Use cases where it doesn't work great:

  • Tasks where you're flipping between a split task and a single-view task. For example, if you want to switch from working on a split task (with two documents side-by-side) to watching a video, you'll need to end split mode

3. Multi-Select

In the same way that you can select multiple files by CMD+Clicking in Finder, we could allow you to select multiple tabs to enter a temporary split view.

Use cases where it works great:

  • you temporarily want to view two tabs side by side
  • Scales well to >2 panes

Use cases where it doesn't work great:

  • you want one tab to be visible in a persistent sidebar (e.g. note-taking, watching a video, or a meeting)
  • you want two tabs to always appear in a split view together

4. Duplex Mode

Allow users to switch to a mode where we always show them two tabs at once: the current tab and the next tab. If you click a tab in the sidebar, the tab you clicked appears in the left pane, and the next tab appears in the right pane.

Pros:

  • Pretty intuitive
  • Switching in and out of the mode is a single action which can be done easily with a keyboard shortcut
  • Scales nicely to >2 panes

Use cases where it works great:

  • You need to hop between many different tabs in split view
  • A master/detail interaction, e.g. viewing a feed or news aggregator in the left pane while viewing content in the right pane

Use cases where it doesn't work great:

  • Tasks where you're flipping between a split task and a single-view task. For example, if you want to switch from working on a split document to watching a video, you'll need to end split mode
